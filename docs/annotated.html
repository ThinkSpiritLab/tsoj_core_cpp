类列表
这里列出了所有类、结构、联合以及接口定义等，并附带简要说明:
 CCompileConfigConfig 的子类，将其父类参数设定为编译时需要的配置参数
 CConfig运行配置类，根据具体运行需求提供所需的子进程运行限制条件与子进程运行环境
例如：运行时间限制，运行空间限制，子进程调用系统函数的权限等
 CContestUpdateJob竞赛类，定义了其独特的 update_solution 操作以及竞赛所需的统计非 AC 提交数等功能 同时，该类负责维护所有 contest 相关的 source, compile_info, solution, user_problem 等表
 CCourseUpdateJob课程练习类，定义了其独特的 update_solution 等操作。针对课程，此类还维护 user_problem 表中 cid 非空的数据。 该数据与普通的 user_problem 记录独立。即同一个 u_id 与 p_id 的数据也根据 cid 不同而不同。竞赛中用户的提交信息也记录 到 course_user 表中，在普通练习未 AC 的情况下，课程中的提交与 AC 会相应增加或更新到 user 表中。该 course_user 表也由此类维护。
 CExecuteArgs执行 exec 族的命令行参数。在原本的 Unix 要求中，exec 族函数的命令行参数末尾必须以一个空指针结尾， 这显然十分丑陋不够优雅，也晦涩难读。基于此目的，本处使用了一个类将它封装了起来。
 CExerciseUpdateJob普通练习类，定义了其独特的 update_solution 操作，即增加特有的 solution 记录。 而维护的 user 表、user_problem 表与 problem 表，均使用基类的定义完成。
 CExerciseUpdateJobBaseExerciseUpdateJob 与 CourseUpdateJob 的虚基类，定义了解题状态查询与更新提交通过数等通用部分。
 Cformat_helper
 CJobBaseJudgeJob 与 UpdateJob 的基类， 存有 Job的基本信息，并规定了所有 Job 必须遵循的函数接口。
 CJobHandleException自定义的 Job 异常，继承于 std::exception，专用于 Job 处理中的异常
 CJudgeJobJobBase 的子类，用于实现提交代码的判题功能。
 CLog_level_traits
 CLog_level_traits< LogLevel::LEVEL_DEBUG >
 CLog_level_traits< LogLevel::LEVEL_FATAL >
 CLog_level_traits< LogLevel::LEVEL_INFO >
 CLog_level_traits< LogLevel::LEVEL_WARNING >
 CMysqlEmptyResException
 Cprocess
 CResult
 CRunningConfigConfig 的子类，将其父类参数设定为运行时需要的配置参数
 CSolutionDetails
 CUpdateJobBaseJobBase 的子类，同时是所有类型的 update 类的基类。
